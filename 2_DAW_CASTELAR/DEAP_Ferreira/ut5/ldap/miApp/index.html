Proyecto de Configuración de Servicios con Docker

Arquitectura basada en contenedores con Apache, LDAP y Traefik.

Características

  • Seguridad en Apache: Desactivación de información sensible en las cabeceras.
  • Autenticación LDAP: Protege el acceso a contenidos mediante autenticación.
  • Gestión de subdominios: Utiliza DuckDNS para subdominios dinámicos.
  • Traefik: Proxy inverso con balanceo de carga integrado.

Servicios

  • Traefik: Proxy inverso que gestiona rutas y balanceo de carga.
  • Servidor PHP-Apache: Servidor web con soporte PHP y seguridad avanzada.
  • Servidor LDAP: Gestiona usuarios y autenticación.
  • phpLDAPadmin: Interfaz gráfica para la administración LDAP.
  • DuckDNS: Gestión automática de subdominios dinámicos.

Archivos Importantes

  • docker-compose.yaml: Define servicios y dependencias.
  • apache-security.conf: Configuraciones avanzadas de Apache.
  • Volúmenes:
    • ldap-data: Datos persistentes del servidor LDAP.
    • ldap-config: Configuraciones del servidor LDAP.
    • duckdns: Configuración de DuckDNS.
    • miApp: Archivos de la aplicación servida por Apache.

Instrucciones de Uso

  1. Clonar este repositorio en tu sistema local.
  2. Configurar las variables de entorno en el archivo .env:
    3. LDAP_ORGANISATION=Ejemplo organizacion
LDAP_DOMAIN=ejemplo.com
LDAP_ADMIN_PASSWORD=Contraseña_Admin
SUBDOMAIN=tu-subdominio
TOKEN=tu-token-de-duckdns
  3. Cargar un archivo PHP o index en la carpeta miApp.
  4. Levantar los servicios con docker-compose up -d.
  5. Acceder a los servicios:
    • phpLDAPadmin: http://phpldapadmin.<tu-subdominio>
    • Aplicación web: http://ldap.<tu-subdominio>
  6. Acceder a la documentación y soporte de cada servicio en sus páginas oficiales.
